# DEV-DIARY — iBrain 开发日志

> 像开发者日记：每次有进展就加一条，**带日期**。  
> 不是 changelog 的替代（版本号仍看 [ROADMAP](../ROADMAP.md)），而是记录 **为什么做、学到了什么、下一次想试什么**。

---

## 怎么用这份文档

| 谁 | 怎么用 |
|----|--------|
| 你（作者） | 每次贡献后加一节 `## YYYY-MM-DD`；考试周空窗也如实写「暂停原因」 |
| Agent（我） | 你要新 Skill / MD 时：把你的想法收成 **Abstract**，写入 Skill 文首或文末，并在本日记 + [SKILL-ABSTRACTS](./SKILL-ABSTRACTS.md) 各记一条 |
| 读者 / 老师 | 快速扫日记看项目在往哪走；看 Abstracts 决定用哪个 Skill |

### 每条日记建议结构（可短写）

```markdown
## YYYY-MM-DD

**做了什么：** …
**为什么：** …
**客观原则：** （如何避免「只会拼凑输出」）
**下次：** …
```

### Skill Abstract 约定

每个新 Skill 在 `SKILL.md` 里保留一段 **`## Abstract`**（文首 frontmatter 后、或文末均可），用 5–8 行说清：

1. **帮谁** — 学生 / 老师 / 提问者  
2. **解决什么客观问题** — 不是「讲得更好听」，而是可检验的行为改变  
3. **不做什么** — 反僵化 / 不诊断 / 不整章讲义  
4. **一句话怎么触发**

摘要索引统一维护在 [SKILL-ABSTRACTS.md](./SKILL-ABSTRACTS.md)。

## 2026-07-21

**做了什么：**

- 新增 [INTERACTION-PROTOCOL.md](./INTERACTION-PROTOCOL.md) — 交互逻辑手册（怎么想、怎么答、怎么记）
- 新增 [community/](../community/) — 教学法贡献区（TEMPLATE + 示范「间隔提取因果链」）
- 本日记 + [SKILL-ABSTRACTS.md](./SKILL-ABSTRACTS.md) 起步（v0.3.1 → 日记体系）

**为什么：**

- 考试后约 5 天没有贡献；今天不想只「再堆一个辅导 Skill」
- 和班主任 / 心理老师聊过：兴趣在心理 / 认知科学，希望以后把概念做成**真能帮学生**的小工具；校内落地不急，先把「交互」和「方法图书馆」做对
- 希望教学交互 **少僵化**：不只要合成资料输出，还要客观地帮助学生做计划、帮老师有效教、帮大家有效提问

**客观原则（今天定调）：**

| 对象 | 客观目标 |
|------|----------|
| 学生 | 有可检验计划与练习，不是听完讲义 |
| 老师 / 贡献者 | 方法可复用、有「不适用」边界 |
| 提问者 | 话术清楚 → 触发对的 Skill（见 HOW-TO-ASK） |

**下次：**

- 新 Skill 时同步写 Abstract + 日记一条  
- 可选：`method-apply`（引用 community/methods）  
- 更远期：校内原型（需老师边界）
