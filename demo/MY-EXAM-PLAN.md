# MY-EXAM-PLAN — 我的考试复习看板

> 作者自用（可公开仓库里保留结构；具体分数/学校名可自己打码）。  
> 动机：课表冲突导致部分课无法在校内跟完 → 自学时**信息断在「上一年课堂」** → 复习无效。  
> 目标：明年 **新课 + 重考课程** 用 iBrain 交互做「可接上的复习」，而不是再堆讲义。

---

## 背景（给未来的自己 / 展示用）

- 今年：部分课程因冲突未在学校完整修读，主要靠自学  
- 结果：复习时缺少连贯课堂信息，效果差（具体分数不在此公开细写）  
- 明年：新课 + 重考已修课程；会有大小考试 → **需要可交互、可写回的复习方式**  
- 短期：先不做独立网站 / App；用 **Cursor + 本仓库 Skills + 本看板** 落地

## 本周规则（客观、短）

1. 每次复习 **先写卡点 + 分钟数**，禁止只说「帮我复习」  
2. 每次结束必须有 **明天第一步**（写进 `sessions/`）  
3. 一次只打通 **1 个断点**；不求一天补完全年缺课  
4. 资料只补 gap；已 solid 的不重讲  

---

## 和 Agent 开始今天的复习

把科目填进括号后发送：

```text
用 study-coach：科目（____）。
背景：去年/今年自学时信息断在（____）。
今天只要（15/25/40）分钟。
请：标 1 个最高优先级断点 → 1 个可检验练习 → 写回「明天第一步」。
不要整章，不要模板。
```

结束后：复制 [`sessions/TEMPLATE-session.md`](./sessions/TEMPLATE-session.md)  
另存为 `sessions/YYYY-MM-DD-科目短名.md`，填入对话要点。
