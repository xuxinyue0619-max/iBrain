# Community — 有效教学法贡献区

欢迎把**真正帮到过学生**的方法写成 Markdown，通过 Pull Request 贡献到本仓库。

目标：让 iBrain 不只是「作者写的 Skill」，而是逐渐变成一个 **可检验的教学法小图书馆**。Agent 可以引用这些方法，但**不得把它们当成死模板填空**（见 [INTERACTION-PROTOCOL](../docs/INTERACTION-PROTOCOL.md)）。

---

## 为什么要贡献

- 课堂 / 辅导 / 自习里试过有效的步骤，值得被复用
- 心理学、认知科学里的概念（间隔提取、费曼、误解纠偏等）可以变成**可操作的短流程**
- 以后若做成校内小工具，这里会是「方法原料库」；编程同学与艺术同学可以各自引用同一条方法做不同产品

**现在：** 只收 Markdown 方法条目。  
**以后（需学校与老师同意）：** 再考虑网站、心理支持工具、真实数据——不在本目录范围。

---

## 怎么贡献（3 步）

1. Fork 本仓库，或直接开 PR  
2. 复制 [`TEMPLATE.md`](./TEMPLATE.md) → `methods/<短横线英文名>.md`  
3. 填完整并提交 PR；标题建议：`method: add <方法名>`

### 命名

```text
methods/spaced-retrieval-chain.md
methods/feynman-gap-check.md
methods/misconception-contrast.md
```

小写英文 + 短横线；一个文件 = 一条方法。

---

## 收录标准（必须满足）

| 必须有 | 说明 |
|--------|------|
| 适用卡点 | 学生具体卡在哪，而不是「复习生物」 |
| 步骤 | ≤ 7 步，可在 15–45 分钟内做完 |
| 可检验产出 | 口述 / 一题 / 一张图 / 一段写回，能判断「做完了没有」 |
| 不适用场景 | 什么情况下别用这个方法 |
| 来源 | 课堂经验 / 文献概念 / 自创；诚实标注 |

### 不收录

- ❌ 纯理论长文、无步骤
- ❌ 「刷 100 道题」这类不可检验口号
- ❌ 医疗/心理**诊断**或危机干预流程（请走学校咨询师；本库只谈学习方法）
- ❌ 整本讲义复制

---

## 隐私与安全

- 不要写真实同学姓名、学号、成绩单、聊天记录原文  
- 示例用虚构或脱敏  
- 涉及情绪困扰时：方法可谈「学习压力下的时间切片」，但须写明 **转介老师/咨询师**，AI 与方法文件不做替代治疗

## 审 PR 时看什么

1. 是否按 TEMPLATE 填全  
2. 步骤是否可执行、产出是否可检验  
3. 是否写了不适用场景  
4. 是否无真实个人隐私

## 和 Skill 的关系

```text
community/methods/*.md  →  方法原料（人贡献）
skillpacks/.../SKILL.md →  Agent 何时调用、如何诊断/写回
docs/INTERACTION-PROTOCOL.md →  交互顺序与反僵化
```

贡献方法 ≠ 改 Skill。想改 Skill 请另开 PR 并说明动机。
